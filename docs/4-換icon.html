Part 4 — 換圖示

怎麼換掉
App 的圖示？

想把桌面上那顆圖示換成自己喜歡的圖，其實不用改任何程式碼，重點在「同檔名覆蓋」加上兩個容易忘的收尾步驟。

圖示由四個檔組成

App 顯示的圖示不是一張圖，而是 icons/ 裡的四個 PNG，分別給不同平台用。換圖就是把這四個檔換掉：

icon-180.png180×180，iOS 加到主畫面用（在 index.html 的 apple-touch-icon 引用）
icon-192.png192×192，Android 圖示（在 manifest 引用）
icon-512.png512×512，Android 安裝／啟動畫面
icon-512-maskable.png512×512，可遮罩版，內容置中讓 Android 怎麼裁都不切到重點

最簡單做法：同檔名覆蓋

只要準備好新圖，用一模一樣的檔名和尺寸覆蓋掉這四個檔就好。這樣 manifest.webmanifest 和 index.html 裡的路徑完全不用改 —— 它們指向的是檔名，內容換了路徑還是對的。

製作要點：

  • 準備一張正方形、至少 512×512 的來源圖（越大越好，例如 1024），再縮出各尺寸。
  • 背景建議不透明，配合深色底 #0c0d12 比較統一。
  • 四張要用同一張來源，否則 iPhone 和 Android 會顯示不同圖。
maskable 那張要留白：Android 各家手機圖示形狀不同（圓、方、水滴），系統會去裁切。所以 icon-512-maskable.png 要把重點縮在中間約 80%，四周補背景色，這樣怎麼裁都不會切到主體。

用一張大圖一次產生四個尺寸

不想開繪圖軟體一張張匯出，在 Windows 上可以直接用 PowerShell 內建的 System.Drawing 縮圖。把來源放成 1024×1024，跑一段腳本就好（前三張直接縮放，maskable 縮到 80% 置中加深色底）：

# 來源：一張 1024×1024 的 PNG
Add-Type -AssemblyName System.Drawing

function Save-Resized($src, $out, $size) {
    $img = [System.Drawing.Image]::FromFile($src)
    $bmp = New-Object System.Drawing.Bitmap($size, $size)
    $g = [System.Drawing.Graphics]::FromImage($bmp)
    $g.InterpolationMode =
        [Drawing.Drawing2D.InterpolationMode]::HighQualityBicubic
    $g.DrawImage($img, 0, 0, $size, $size)
    $bmp.Save($out, [Drawing.Imaging.ImageFormat]::Png)
    $g.Dispose(); $bmp.Dispose(); $img.Dispose()
}

Save-Resized "source.png" "icons/icon-180.png" 180
Save-Resized "source.png" "icons/icon-192.png" 192
Save-Resized "source.png" "icons/icon-512.png" 512
# maskable 另外把圖縮到 0.8 置中、四周填 #0c0d12
後端類比：這就像用一張原始大圖，在不同 endpoint 回傳不同解析度的縮圖（image resizing）。來源只留一份高解析的，輸出各尺寸由程式產生，才不會每張都要手動維護。

兩個一定要做的收尾

圖換好了，但你重新整理網頁常常還是看到舊圖 —— 因為被「快取」住了。有兩層快取要處理：

  1. 更新 Service Worker 版本號。
    打開 sw.js，把 const CACHE = "timer-v2" 往上加成 "timer-v3"。版本號沒變，瀏覽器就認定內容沒變，會一直回舊圖。
  2. 把圖示從主畫面移除再重新加入。
    iOS／Android 對「已加到主畫面」的圖示會另外快取一份，連改了版本號都未必馬上更新。移除 → 重新整理網頁 → 再「加入主畫面」一次最保險。
為什麼改版本號就會更新？Service Worker 的離線快取像一個有版本的本地快取層（概念近似 cache key）。改了 key，舊快取整批失效，瀏覽器才會重新去抓新檔。這是 PWA 最常見的坑：改了檔案卻忘了動版本號，使用者永遠看不到更新。
順帶一提：當來源用的高解析原圖（例如那張 1024 的 .jpg）不會被 App 用到，manifest 和 index.html 只引用 .png。它放進 repo 只是多餘的素材，可以刪掉，或加進 .gitignore（例如 icons/*.jpg），自己在電腦別處留一份即可。
