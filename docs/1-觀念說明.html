Part 1 — 觀念

icon 與 manifest
是做什麼用的？

把計時器變成「桌面上可以點的 App」會用到幾個檔案，這頁說明它們各自的角色，以及怎麼串在一起。

icon：App 的「長相」

那幾張 png，就是加到主畫面後，出現在 iPhone 桌面上的那顆圖示。沒有它，系統只會抓一張網頁縮圖當圖示，通常很醜、看不出是什麼。

準備多種尺寸，是因為不同地方用不同大小，系統會挑最合適的那張，避免硬放大而模糊：

icon-180
iOS 專用
加到主畫面抓這張
192 / 512
Android 與
安裝畫面、啟動畫面
512 maskable
可被裁成圓/方
內容留在中央安全區

最後一種「maskable（可遮罩）」是因為 Android 各家手機圖示形狀不同（圓、方、水滴），系統會去裁切。這版把內容縮在中央，怎麼裁都不會切到重點。

manifest：App 的「身分證」

manifest.webmanifest 是一份設定檔，告訴瀏覽器：這個網頁要被當成 App 安裝時，叫什麼名字、用哪個圖示、要怎麼顯示。

後端類比：它就像 Spring Boot 的 application.yml，或 Android 的 AndroidManifest.xml —— 一份描述應用程式 metadata 的設定檔。

每一欄的作用：

name安裝畫面顯示的完整名稱
short_name桌面圖示下方的短名稱（太長會被截斷）
start_url點圖示打開時，從哪個網址啟動
display最關鍵的一行，見下方
orientation鎖定方向，portrait 是直向
background_color啟動瞬間、內容還沒載完時的底色
theme_color系統 UI（狀態列等）的色調
icons指向上面那些圖示檔

其中最關鍵的是 "display": "standalone"。它決定你點桌面圖示打開時長什麼樣：

standalone → 全螢幕、沒有 Safari 的網址列和工具列，像獨立 App。
browser → 打開後還是像在 Safari 裡開網頁，有網址列。

你要的「像 App 一樣」體驗，就是這一行給的。

三者怎麼串起來？

用一條鏈來看就清楚了，各司其職：

網頁本體
index.html
head 裡寫一行 link 指向 manifest
<link rel="manifest">
App 身分設定檔
manifest.webmanifest
定義名稱、顯示模式，再指向圖示
"icons": [ ... ]
桌面圖示外觀
icon-*.png
不同尺寸，系統挑合適的用

少了 manifest，瀏覽器就不認為這是個可安裝的 App，「加入主畫面」只會存成普通書籤，而不是 App。

還有第四個角色：sw.js（Service Worker）管的是「離線快取」，跟外觀、身分無關 —— 它負責「沒網路也能開」。概念上像後端的快取中介層（cache-aside）：請求先查快取，沒有才連網路。
