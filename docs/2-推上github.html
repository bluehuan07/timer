① 觀念 ② 推上 GitHub ③ 在 iPhone 安裝
Part 2 — 部署

推上
GitHub Pages

把計時器放到網路上，拿到一個免費的 HTTPS 網址。整個過程不用裝任何工具，在 GitHub 網頁上點一點就好。

要上傳的檔案
  • index.html
  • manifest.webmanifest
  • sw.js
  • icons/ （內含 icon-180 / 192 / 512 / 512-maskable 四張）
注意：讓 index.html、manifest.webmanifest、sw.js 直接躺在 repo 根目錄，4 張圖示保持在 icons/ 資料夾裡一起上傳。不要把最外層那個資料夾也一起傳上去——路徑層級錯了，圖示和 manifest 會抓不到。
1

建立 repository

到 GitHub 按右上角 New repository。取個名字（例如 timer），務必設為 Public（免費版 Pages 需要公開），按建立。

2

上傳檔案 — 方法 A：網頁拖檔

在 repo 頁面點 Add file → Upload files，把檔案全部一起拖進去（icons/ 連資料夾一起拖），最後按 Commit changes。改一次拖一次，適合偶爾更新。

方法 B：用 IntelliJ 直接推（推薦給後端）

專案本來就在 IdeaProjects 底下，用 IntelliJ 內建的 Git 整合最順，之後改檔也是一鍵 push，不必每次回網頁拖檔。

先說一個常見卡點：剛用 IntelliJ 開這個資料夾時，找不到「Git」選單是正常的。資料夾還沒納入版控前，頂部只有 VCS 選單；等你做完下面第 2 步（等於 git init），Git 選單才會出現。
  1. 登入 GitHub 帳號（只需一次）：
    Settings → Version Control → GitHub → 按「+」 → 用瀏覽器授權登入。
  2. 啟用版控：頂部 VCS → Enable Version Control Integration… → 選 Git。這步等於 git init，做完「Git」選單就會冒出來。
  3. 建立 commit：按 Ctrl+K 打開 Commit 面板 → 勾檔案、寫訊息 → Commit。（.gitignore 已幫你排除 .claude/ 與 IDE 設定，會自動略過。）
  4. 接上遠端 repo：Git → Manage Remotes → 按「+」 → 貼上你網頁建好的網址 https://github.com/你的帳號/timer.git。
  5. 推送：Git → Push（Ctrl+Shift+K）。

不管用方法 A 還 B，推完程式碼還沒上線——都要再回網頁做下面第 3 步開啟 Pages，才會生出可用的網址。

3

開啟 Pages

進入 repo 的 Settings → Pages

在 Source 選 Deploy from a branch，Branch 選 main、資料夾選 / (root)，按 Save。

4

等待並取得網址

等一兩分鐘後重整這個頁面，上方會出現你的網址：

https://你的帳號.github.io/timer/

這就是可以分享給任何人的公開網址。下一頁教你怎麼把它變成 iPhone 上的 App。

給後端的提醒：之後若改了任何檔案重新上傳，記得把 sw.js 裡的 timer-v1 改成 timer-v2，使用者重開 App 才會拿到新版，否則會一直吃舊的離線快取。這是 PWA 最常見的坑。
